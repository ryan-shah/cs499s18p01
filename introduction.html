Introduction

Fishback Management and Research Inc. (sometimes refered to as ODDS), a stock trading company, has many different PHP and Bash scripts that they run in sequence every night to keep their databases and customer facing products up to date and working properly, in addition to related maintenance tasks. They are having issues with the length of time this process takes as each script is executed one at a time, right after each other. As a result, this means the total run time is several hours. Our job is to find a solution to this problem, in additon to creating a web based portal to manage these scripts and which ones are run each night.

Our solution is Spydr's Threads (Spydr). This will solve the time issue by intelligently running groups of scripts in parallel threads, while also enuring dependencies are satisified and nothing is run out of order. This utility will be accessed by ODDS server administrators via a web portal that allows them to create/edit/remove scripts in addition to being able to add/remove them from the schedule of scripts to run each night. They will also be able to create multiple schedules to run at different times and/or days.
