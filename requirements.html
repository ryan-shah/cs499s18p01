2.1

Technical Requirements (in order of priortiy)

  1. PHP 7.0 Webpage and MYSQL Database of programs
  2. Run shell-executable scripts in parallel on a Linux server
  3. Identify dependencies of programs and ensure they are run first
  4. Ability to schedule how frequently this series of programs are run
  5. Process statistics and tracking during execution
  6. Dynamic load distribution of programs over server resources based on user defined specifications
  7. Add/Edit list of known programs

2.2

Effort and size estimation

The effort required to make a website which has all of the basic functionality which our customer wants (i.e. something that can edit a database, able to call an external runner program) is not going to be very high after we learn more about the Laravel framework. All of us have some experience in web development, so we won't have to take any time to get acquainted with the foundations of web programming. If we had to estimate, the basic functionality will take no more than two to three weeks to implement.
Another component to this program is the runner. We plan on writing this in C/C++, as it has to be high-performance for running memory and CPU intensive programs with the least overhead possible. We are still not entirely sure how we will go about doing this. This program will likely not be very long, but integrating it with the website and all of the programs that our customer is already running might. We still don't have access to everything we need to get a good estimation on the complexity required on this part.
Another crucial component is the database. Before we create anything, we will have to consider design choices which minimize memory usage and redundancy. Again, all of us have experience designing and creating databases, so most of the time spent here will be on coming up with good design choices.
There are several goals that we could continue to pour time into and continually refine. Storing performance metrics will not be difficult to do initially; there are a few terminal commands that will give this information to us at any time. However, creating an algorithm to optimize the future nightly runs will not be trivial. The frontend of the website could also always be refined, giving consideration to design choices that are less resource-intensive while producing the same output.
