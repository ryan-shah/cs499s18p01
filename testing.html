Testing

  1. Unit Testing

    2. For unit testing we will perform the following checks on the different components of our project:

    • Website

      1. Fetches Data from Database
      2. Updates Data in Database when form is submitted (add/edit/remove scripts/schedules)
      3. Displays forms properly
      4. Writes Schedule to config file when form is submitted
      5. Reads updated values from log file when schedule is run

    • C++ Program

      1. Read data from config file
      2. Check current time and know when to run
      3. Check dependencies of script to see if script can run
      4. Check available memory to see if script can run
      5. Track running script and record its memory/runtime statistics
      6. Know when script finishes so dependencies can run
      7. Write to log files most recent execution statics for each script

  2. Integration Tests

    3. These tests how our modules interact with each other so the following tests will check to make sure our connections in the process are working as intended.
    1. Add/remove script/schedule to database and ensure it appears on website
    2. Add/remove script/schedule to website and ensure it appears in database
    3. Change runtime of schedule in database and ensure it appears on website
    4. Change runtime of schedule on website and ensure it changes in config file and C++ program starts on time
    5. On completion of C++ program ensure updated runtime statistics appear in database

  3. System Testing

    4. There will not be a lot of different systems that need to be tested as this application will only run on the Linux server it was designed for. That being said, we will test the website's compatiblity with the following browsers that we know our customer's use:
    • Firefox 59.0.2
    • Chrome 65.0.3325.185
    • Safari 10.1

  4. Acceptance Testing

    5. For acceptance testing, we will install Spydr onto the customer's server and they will add several "dummy scripts" that they use for testing into our database via the webpage. They will set up a schedule via this web portal as well and then see if it runs successfully and the total execution time is decreased and everything executed successfully. This will be the final test of our program before delivery.
